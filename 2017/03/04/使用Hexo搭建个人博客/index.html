使用hexo搭建个人博客

环境准备

Node.js

到Node.js官网下载最新版的Node.js，一路安装即可。

Git

Windows下要使用很多Linux/Unix的工具时，需要Cygwin这样的模拟环境，Git也一样。Cygwin的安装和配置都比较复杂，就不建议你折腾了。不过，有高人已经把模拟环境和Git都打包好了，名叫msysgit，只需要下载一个单独的exe安装程序，其他什么也不用装，绝对好用。
msysgit是Windows版的Git，从https://git-for-windows.github.io下载，然后按默认选项安装即可。

Sublime

开始搭建

Github Pages

Github Pages免费的静态站点，其特点：

  • 免费托管
  • 自带主题
  • 支持自制页面

创建Github Pages比较简单，只要你有一个github账号在创建一个仓库就行了，但是这个仓库是有规则的，其格式必须为：your-account.github.io。然后根据提示一直下一步即可，非常简单。

Hexo

Hexo 是一个快速、简洁且高效的博客框架。

  • 超快速度：Node.js 所带来的超快生成速度，让上百个页面在几秒内瞬间完成渲染。
  • 支持Markdown：Hexo 支持 GitHub Flavored Markdown 的所有功能，甚至可以整合 Octopress 的大多数插件。
  • 一键部署：只需一条指令即可部署到 GitHub Pages, Heroku 或其他网站。
  • 丰富的插件：Hexo 拥有强大的插件系统，安装插件可以让 Hexo 支持 Jade, CoffeeScript。

之前所有必备的应用程序安装完成后，即可使用npm安装Hexo。

$ npm install -g hexo-cli

NexT

玩博客，换主题是必不可少的。Hexo的默认主题是landscape。Hexo提供多种风格的主题，可通过以下方式自行下载。我最喜欢的主题是NexT，简单大方，有三种不同的风格。

$ cd your-hexo-site
$ git clone https://github.com/iissnan/hexo-theme-next themes/next

使用Hexo建站

初始化

安装 Hexo 完成后，请执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件。

$ hexo init <folder>

新建完成后，指定文件夹的目录如下：

.
├── _config.yml # 网站配置信息
├── package.json # 应用程序信息
├── scaffolds # 模板文件夹，Hexo在新建文章时，会根据模板文件夹填充Markdown信息
├── source # 资源文件夹，除_post和_drafts文件夹之外,其他的文件将会被解析到public文件夹
|   ├── _drafts # 草稿
|   └── _posts # 已发布
└── themes # Hexo会根据主题来生成静态页面

新建文章

新建一篇文章，文章会被保存到 Your_Project_File/source/_posts/Article_title 。

$ hexo new [layout] <Article_title>

Hexo 有三种默认布局：post、page 和 draft 。

您可以在命令中指定文章的布局（layout），默认为 post，可以通过修改 _config.yml 中的 default_layout 参数来指定默认布局。

在新建文章时，Hexo 会根据 scaffolds 文件夹内相对应的文件来建立文件，例如：

$ hexo new [post] "Test"

在执行这行指令时，Hexo 会尝试在 scaffolds 文件夹中寻找 post.md，并根据其内容建立文章。

如果文章的标题包含空格，则必须使用“ ”。

<post.md>
---
title: {{ title }}
date: {{ date }}
tags: 
categories： 
---

<Test.md>
layout: '[post]'
title: Test
date: 2017-03-04 11:52:13
tags:
---
You can write something here.

生成静态页面

执行如下命令，生成静态页面至 hexo\public\ 目录。

hexo generate

命令必须在init目录下执行，否则不成功，但是也不报错。

###启动本地服务器

启动服务器。默认情况下，访问网址为： http://localhost:4000/。

$ hexo server

若端口被占用，服务器无法启动。需要我们进行重设端口。

$ hexo server -P 5000

使用Next

启用主题

当安装完成后，打开 站点配置文件， 找到 theme 字段，并将其值更改为 next。

theme: next

主题设定

选择Scheme

Scheme 是 NexT 提供的一种特性，借助于 Scheme，NexT 为你提供多种不同的外观。同时，几乎所有的配置都可以 在 Scheme 之间共用。目前 NexT 支持三种 Scheme，他们是：

  • Muse - 默认 Scheme，这是 NexT 最初的版本，黑白主调，大量留白
  • Mist - Muse 的紧凑版本，整洁有序的单栏外观
  • Pisces - 双栏 Scheme，小家碧玉似的清新

Scheme 的切换通过更改 主题配置文件，搜索 scheme 关键字。 你会看到有三行 scheme 的配置，将你需用启用的 scheme 前面注释 # 即可。

主题配置文件为：Your_Project_File/themes/next/_config.yml

#scheme: Muse
#scheme: Mist
scheme: Pisces

设置语言

编辑 站点配置文件， 将 language 设置成你所需要的语言。建议明确设置你所需要的语言，例如选用简体中文，配置如下：

站点配置文件为：Your_Project_File/_config.yml

language: zh-Hans

添加标签和分类页面

  1. 在终端窗口下，定位到 Hexo 站点目录下。使用 hexo new page 新建一个页面，命名为 tags ：

    $ cd your-hexo-site
$ hexo new page tags

  2. 编辑刚新建的页面，将页面的类型设置为 tags ，主题将自动为这个页面显示标签云。页面内容如下：

    title: 标签
date: 2014-12-22 12:39:04
type: "tags"
---

  3. 在菜单中添加链接。编辑 主题配置文件 ， 添加 tags 到 menu 中，如下：

    menu:
  home: /
  archives: /archives
  tags: /tags

  2. 创建站点，填写站点相关信息。多说域名这一栏填写的即是你的 duoshuo_shortname，如图：
  3. 创建站点完成后在主题配置文件中新增 duoshuo_shortname 字段，值设置成上一步中的值。
  4. 在站点配置文件中正确设置 url 为你站点的域名。

添加阅读次数统计

不蒜子可以给任何类型的个人站点使用，如果你是用的Hexo，打开themes/next/layout/_partial/footer.ejs添加以下脚本即可。

<script async src="//dn-lbstatics.qbox.me/busuanzi/2.3/busuanzi.pure.mini.js">
</script>

只需要复制相应的html标签到 themes/next/layout/_partial/footer.ejs 添加即可。可以随意更改不蒜子标签为自己喜欢的显示效果。

<span id="busuanzi_container_site_pv">
    本站总访问量<span id="busuanzi_value_site_pv"></span>次
</span>

Enjoy it!
